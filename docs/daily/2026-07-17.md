## NENE2 表記のリポリンク化（DB内容変更・施主GO）

- 施主「NENE2 の部分、GitHub リポ URL にリンクした方がいい」→ 調査で**より重要な発見**: /products 証拠棚の「NENE2 は公開フレームワーク →」カードのリンク先が **profile で、リポ本体でなかった**（主張と証拠の不一致）。
- 実施（5箇所・ページ毎スコープの REPLACE・置換数 1/1/1/1 検証済み）: カードのリンク先是正＋各ページ初出（/・/products・/services・/company）の文中リンク化。既存外部リンクの作法踏襲（明色 bg=#D64525・暗色 bg=#E0603F・下線なし bold）。2回目以降の出現・JSON-LD/メタは非対象。リポは公開・description 整備済み・07-14 更新を事前確認。
- ★教訓（30分ハマり候補だった）: **この DB の text_fields を mysql クライアント経由で REPLACE するときは `SET NAMES utf8mb4` 必須**。無いと日本語リテラルが charset 変換で不一致になり REPLACE が silent no-op（LIKE '%NENE2%' など ASCII は当たるので気づきにくい）。07-16 の footer 置換が通ったのは CHAR(10) 連結と部分 ASCII アンカーの偶然。以後の内容変更 SQL はテンプレ先頭に SET NAMES を固定。
- ★もう1つ: DB 内の bespoke html は**クォートが `&#039;` エンティティ**（sanitizer 出力）。rendered HTML からアンカー文字列を採ると不一致になる — 置換文字列は必ず DB 生値（SUBSTRING）から採る。
- README が見込み客から1クリックになった件は統合リナへ相談中（(a) NENE2 リナ依頼 /(b) hub 裁定 /(c) records で read-only 監査、推し=(c)→(a)）。
