# 市場ペルソナ討論（補遺）— 計測機構と UTM の要否（2026-06-26）

第5回（#581）の集約を踏まえた**トピック特化の短時間会合**。「すでに入っている GA/計測機構の上に、UTM リンクビルダー等の "GA 導入機構" を新たに作るべきか」だけを問うた。フル判定ラウンドではない。

## 全ペルソナに共有した現状（ブリーフィング）

1. **計測機構は完成・本番反映済**（PR-A1 #583 / PR-A2 #584）。管理画面で **GA4 測定 ID か GTM コンテナ ID** を入れると、公開サイトが CSP 緩和（ID 設定時のみ）＋**Consent Mode v2** 既定（EU 安全側 denied）＋同意バナー＋SPA ルート `page_view` を自動で動かす。本番は現在 **OFF**（ID 未設定）。
2. **UTM は GA に ID を入れれば追加実装ゼロで効く**。GA4/GTM が着地 URL の `utm_*` を自動でキャンペーン/流入元ディメンションに取り込み、SPA のクライアント遷移後もセッション帰属は保持される。
3. 議題＝**「UTM リンクビルダー（UTM 付き URL を生成する管理画面ツール）等を新たに作るべきか」**。これは計測そのものではなく "道具/導線" の話。

## 各ペルソナの要点

| ペルソナ | 立場 | 理由 |
| --- | --- | --- |
| SaaS営業 | 見送り | Google の無料 Campaign URL Builder がある以上、内製は堀にならない。自分の壁は今も「移行参照1件＋保守SKU」で UTM ではない。 |
| 中小企業経営者 | 無関心 | そもそも UTM を知らない/使わない層。欲しいのは「GA をどう入れるか」を埋める**設定ガイド＋発火確認**であって、リンクビルダーではない。 |
| 受託/代理店（佐藤） | "嬉しい程度" | クライアント運用で UTM は使うが、スプレッドシート＋無料ツールで足りる。価値はむしろ「客サイトに GA を CSP/同意対応で安全に入れられる」今の機構そのもの（＝解決済）。 |
| 連続起業家 | 最下位 | 差別化ゼロ（無料代替が豊富）。優先度は収益導線/managed。 |
| データ/分析寄り | 限定的 | 唯一の価値は「命名規則を強制し UTM の表記ゆれを防ぐ」ガバナンス。だが小規模 CMS の客層には過剰。 |

## 集約（結論）

| 観点 | 判定 |
| --- | --- |
| ニーズはあるか | **弱い**。UTM 計測自体は GA で既に動く＝ゲートではない |
| 欲しいか | **限定的**（代理店が "嬉しい程度"、非技術者層は無関心） |
| 導入すべきか | **今は No**。無料の Google ツールで代替十分、堀にならない |
| 代替で価値があるとすれば | GA **導入オンボーディング/発火確認 UX**（非技術者が GA を入れられる導線）。ただし優先度は **managed入口/収益SKU の下** |

**一言**：計測の中身（UTM 含む）はもう足りている。足りないのは "計測機構" ではなく "非技術者が GA を設定できる導線" だが、それすら最優先ではない。最優先は依然 **managed入口/収益SKU（事業判断）**。

## 決定

- **GA/計測まわりは現状で確定**（PR-A1/A2 で機構は完成、UTM は GA 標準対応で充足）。**UTM リンクビルダー等の追加実装は当面行わない。**
- 計測周辺で将来検討の余地があるのは「GA 設定オンボーディング」のみ（優先度低・managed/収益 SKU の下）。
