## 決定的洞察
**出荷の方向と「動かないペルソナ群」のゲートが構造的にズレている。** R3以降の全コミット（#561-575）は1本の軸＝「自前Dockerを回せる受託/SME 向けの WXR移行＋SEO保全＋自己完結イメージ」に集中し、これは佐藤を採用へ押し上げ、技術系ペルソナの「工学は本物」評価を固めた。だが製品自身が初期ICPを「Dockerを自走できる層」に明示固定している結果、それ以外の全ペルソナのゲートには物理的に1mmも効いていない：
- 非技術者の入口（signup/managed/turnkey が全リポジトリでゼロ）＝田中/Priya/Sofia/中小経営
- WYSIWYG(#538 OPEN)＝Sofia / 公開i18n(#540 OPEN・SSR は `html lang` ハードコード)＝田中/Priya
- 計測層（CSP `script-src 'self'` 固定で GTM/GA4 ブロック・計測ID Setting 無し）＝Marcus/SaaS営業
- 運用3点（app healthcheck 無し・mysqldump runbook 無し・Apacheログ未配線）＝陳/SRE/山本

そして**全12人を貫く唯一の共通固着＝「公開ドメインの本番実稼働リファレンス・ゼロ」**（検証は使い捨てコンテナのみ、本番モードで初めてCSPバグが出た事実が本番限定の未知を象徴）。工学では潰せない**市場証明の欠落**。

## 判定を動かす一手（投資対効果順）
1. **開発元自身が1サイトを実公開ドメインへ本番デプロイし、数週間無事故で運用したリファレンスを作る** ＝12人ほぼ全員が要求する唯一の共通ゲート。これ1件で条件付き6人の最重要ブロッカーが外れ、複数が今すぐ採用レンジへ。（実ホスティング/DNS はオーナー作業）
2. **README整合＋NENE2 sibling-clone の1行追記**（ゼロコスト×即効）＝評価入口の信頼を即回復し、外部 clone 時の `composer install` 失敗を塞ぐ。コミット1本。R1〜R4で4連続後回し（バス係数懸念を増幅）。状態表記も実態へ（M1-M9→M16+、545→902 tests、:8080→180xx、MCP数）。
3. **本番運用3点の小PR**＝app へ `/health` healthcheck ＋ mysqldump バックアップ/復旧 runbook（理想は cron 配線）＋ Apache access/error を stdout/stderr へ。陳/SRE/山本を一括で解く。

※見送り5人（田中/Priya/Sofia/山本/Marcus）は上記では動かない。公開i18n #540 / WYSIWYG #538 / 計測 allowlist＋計測ID Setting / マネージドSKU・JIS準拠・保守契約 という**別軸（＝ICP拡張）**が要る。最小投資で最大の判定移動を取るなら **「実公開リファレンス1件＋README 1行修正」** に集中すべき。

## 残る主要条件（上位）
1. 公開ドメインでの本番実稼働リファレンス1件（開発元自身か他者・数週間無事故）— 共通ゲート・固着の核
2. 外部開発者向け NENE2 依存の解決（Packagist/VCS pin もしくは README に `../NENE2` sibling-clone 実コマンド）＋ README 状態表記更新 — コミット1本の near-trivial
3. 本番運用3点（app healthcheck ＋ mysqldump runbook ＋ Apacheログ stdout 配線）
4. 非技術者向けマネージド/ターンキー/1クリック導入、もしくは固定費の受託保守先
5. 公開ページへ第三者タグを載せる正規手段（CSP allowlist ノブ＋計測ID Setting＋consent mode v2）
6. 公開サイト多言語化（#540・`html lang`/hreflang/og:locale）
7. 保守契約可能な障害対応窓口・JIS X 8341 準拠表明・WYSIWYG ライブプレビュー(#538)

## 結論
**製品は選ばれたICP1層（自前Dockerを回せる受託/SME）に対しては"採用直前"まで成熟した**（佐藤が実証）。だが他層には設計思想として届かず、市場の過半は据え置き。次の最大レバレッジは**新機能でなく「実公開ドメインのリファレンス」＋「評価入口（README）の整合」**——いずれも工学的にはほぼ完成しており、残るは"実際に出す/書く"こと。
