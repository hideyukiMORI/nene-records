# コンテンツ・ページング ペルソナ会議（2026-06-26）

> 青空文庫(PD小説)の取り込みで「1作品＝1巨大ページ（例 1.95MB HTML・約33万字）」が重く、動画にも地味な件。長文の扱い（ページング）方針を、エンジニア視点に偏らないよう **SEO / コンテンツマーケ / 読者UX / デジタルパブリッシャー** の4ペルソナ（独立評価）に当てた回。
> 実装位置の事前調査は別途実施（公開レコードの SSR/ルーティング/レンダリング配管をマップ済み）。

## 叩いた原案（エンジニア案）
- 1レコードを**レンダリング時に ~1600字 or 章(H2)で分割**、URL `/{type}/{slug}/{n}`、prev/next、`rel=prev/next`、per-page SSR、title にページ番号。
- **全長文の汎用デフォルト**（blog/docs/商品ページ/小説、全テナント）。

## 判定（4/4）

| ペルソナ | 判定 |
| --- | --- |
| テクニカルSEO | adopt-with-changes・**デフォルト不可**・本/章のみ・各章 self-canonical＋固有タイトル |
| コンテンツマーケ | adopt-with-changes・**デフォルト不可**・既定は単一ページ＋sticky TOC＋進捗バー |
| 読者UX(長文/JP) | adopt-with-changes・**本/章のみ**・char切り却下・見出し無→単一スクロール＋進捗＋復帰 |
| パブリッシャー | adopt-with-changes・**デフォルトを反転**・char分割禁止・"丸ごと"の正規URL＋フィード維持 |

## 全会一致
1. **デフォルトでページングしない。** 単一の正規ページが blog/docs/商品ページの既定。ページングは**本/連載限定の opt-in**。
2. **~1600字の機械切りは却下**（文中・ルビ分断／reflow でブックマーク崩壊／薄い重複ページ）。
3. **分割は実章境界(H2/構造)のみ**＝著者の構造に従う。
4. **`rel=prev/next` は索引に効かない**（Google 2019 廃止）＝戦略の土台にしない。

## 分野別の決め手
- **SEO**：canonical が生死。page1 へ canonical→本文がほぼ index 落ち／char切りの self-canonical→薄い重複量産。**本は「各章 self-canonical＋人間可読の固有タイトル」**なら index 面が増えて妥当。被リンク/共有は page1 に集中＝分割は信号分散・クロール浪費。
- **マーケ**：char ページング＝**PV水増しの最古手**（dwell/読了を犠牲）。"次へ"は離脱の扉、ショート動画では**2010年のスライドショー**に見えてブランド毀損。**動画の決め画＝進捗バーが伸びる sticky TOC 付きの"ぬるぬる単一スクロール"**（Medium/Substack/NYT）。
- **読者UX**：char切りは Ctrl-F/コピー/読み上げ/ふりがなを破壊。**見出し無→単一スクロール＋進捗＋"最後に読んだ位置"復帰**。docs は単一ページ必須。
- **パブリッシャー**：**「重い」はレンダリング問題＝lazy/仮想描画で解く、URL分割で解かない**。`/slug/n` を全テナントの URL契約＋分析に焼くと実質不可逆＋PV指標が崩れる。RSS/EPUB/リーダーは"丸ごと"を要求。

## 結論＝改訂仕様
**デフォルト（全コンテンツ）＝単一の正規ページ** `/{type}/{slug}`：
- **sticky TOC（H2/H3）＋読書進捗バー＋アンカー deep-link**（＝読み味・SEO最良・動画の決め画）。
- 超長文の重さは**本文の lazy/仮想描画**で解決（URL は割らない）。

**opt-in「章モード」＝本/連載コンテンツタイプのみ**：
- **実章境界(H2/章マーカー)のみ分割**・char切り禁止。
- 各章＝ページ・**self-canonical＋固有タイトル**（"Page 7"禁止）。章 prev/next＋章TOC＋最後に読んだ位置。
- **"通し読み/view all"の正規単一URL＋フルフィード(RSS)を常に維持**。
- Aozora：importer は章を H2/マーカーで出力。章無し作品は単一ページ（進捗＋lazy）＝char切りしない。

## フェーズ
- **P1**：単一ページ読書UX（sticky TOC＋進捗＋アンカー）＝デフォルト＆動画の決め画。
- **P2**：章モード（本/連載 opt-in・各章 self-canonical＋通し読み正規URL＋フィード）＋ Aozora importer 章対応 → 再 import。
- **P3**：超長文 lazy/仮想描画（perf）＋ EPUB/reader-mode 等。
