### 2. 全文検索 API（Issue #129 / PR #131）✅

**バックエンド:**

- `EntityListCriteria` に `?string $q` を追加
- `PdoEntityRepository::buildCriteriaWhere()` に LIKE + EXISTS サブクエリを追加:
  ```sql
  (e.slug LIKE ? OR EXISTS (
    SELECT 1 FROM text_fields tf
    WHERE tf.entity_id = e.id AND tf.is_deleted = 0 AND tf.value LIKE ?
  ))
  ```
  - JOIN ではなく EXISTS にすることで重複行を排除
- `ListEntitiesHandler` が `?q` クエリパラメータを読み取り `EntityListCriteria` に渡す
- OpenAPI YAML に `q` パラメータを追記

**フロントエンド:**

- `useEntityList` フックに `q` パラメータを追加
- `ManageEntitiesView` に検索ボックス（インクリメンタルサーチ + ✕ クリアボタン）を追加
- `use-manage-entities-page.ts` に `searchQuery` state を追加

**テスト追加（`PdoEntityRepositoryTest`）:**
- `testFindByCriteriaSearchBySlug`
- `testFindByCriteriaSearchByTextField`
- `testFindByCriteriaSearchReturnsEmptyWhenNoMatch`
- `testFindByCriteriaSearchCombinesWithEntityTypeId`

### 3. CSV/JSON エクスポート API（Issue #130 / PR #132 🔄）

**バックエンド:**

- `ExportEntitiesHandler` を新規作成（`GET /api/v1/entities/export`）
  - `format=csv|json`、`entity_type_id=X`、`q=...` をサポート
  - 最大 10,000 件まで出力
  - `findByEntityIds()` による N+1 防止バッチロード
  - CSV: `id, slug, status, published_at, <fieldKey>...` の列構造
  - JSON: `{"items": [...], "total": N}` で `text_fields` をネスト
- `TextFieldRepositoryInterface` に `findByEntityIds(array $entityIds): array` を追加
  - `PdoTextFieldRepository` / `InMemoryTextFieldRepository` を実装
- `EntityRouteRegistrar` を `/api/v1/entities/export` ルートに対応
  - `{id}` ルートより前に登録してリテラル一致を確保
- `EntityServiceProvider` に `ExportEntitiesHandler` を DI 登録
- OpenAPI YAML に `GET /api/v1/entities/export` を追記

**フロントエンド:**

- `build-export-url.ts` ユーティリティ（`entityTypeId`, `format`, `q?` → URL）
- `ManageEntitiesView` に ↓ CSV / ↓ JSON ダウンロードリンクを追加

**HTTP テスト更新（全 8 ファイル）:**
- `EntityRouteRegistrar` の第 7 引数 `ExportEntitiesHandler` を追加

## 技術的なポイント

| トピック | 内容 |
| --- | --- |
| react-refresh | Hook を別ファイルに分離しないと `--max-warnings 0` が通らない |
