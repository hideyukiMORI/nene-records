### 2. 全文検索 API（Issue #129 / PR #131）✅

**バックエンド:**

- `EntityListCriteria` に `?string $q` を追加
- `PdoEntityRepository::buildCriteriaWhere()` に LIKE + EXISTS サブクエリを追加:
  ```sql
  (e.slug LIKE ? OR EXISTS (
    SELECT 1 FROM text_fields tf
    WHERE tf.entity_id = e.id AND tf.is_deleted = 0 AND tf.value LIKE ?
  ))
  ```
  - JOIN ではなく EXISTS にすることで重複行を排除
- `ListEntitiesHandler` が `?q` クエリパラメータを読み取り `EntityListCriteria` に渡す
- OpenAPI YAML に `q` パラメータを追記

- `use-manage-entities-page.ts` に `searchQuery` state を追加

## 課題・所感

**マイグレーション番号衝突のサイレントスキップ**
Phinx は既存バージョンを黙って飛ばす。`phinxlog` に衝突バージョンが残っているとエラーにならないため、スキーマが未適用のまま API が 500 を返す。`migration-new.php` を整備してこの種のミスを構造的に防いだ。

**「型は openapi から」の徹底**
`created_at` の型ズレは、OpenAPI スキーマと `UserDto` 手書き型の乖離が原因だった。codegen に移行することで同種の問題は今後起きない。`npm run codegen` → 型エラー → 直す、のサイクルが機能するようになった。

**DX 投資の実感**
7 本の改善はどれも「1 回」の効果ではなく毎日の作業速度に効く。特に check:fast（storybook ビルドを除いた高速チェック）と MSW ブラウザモードは、バックエンドを起動せずに UI 作業が完結するので、複数プロダクトを並行開発している状況に合っている。
