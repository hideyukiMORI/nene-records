## 課題・所感

**コンフリクト解消の連鎖**  
M5 の PR 2 本が同じ `AppShell.tsx` / `router.tsx` / `i18n` ファイルを触っていたため、マージするたびに次がコンフリクト。「両方の変更を union で残す」という原則を守ることで安全に解消できた。

**エラーパターンの繰り返しを防ぐ参照確認の重要性**  
コメント実装で発生したバグ 3 件（インターフェース名・OpenAPI $ref・ボディパーサー）は、いずれも既存の類似コード（`NavigationItemNotFoundExceptionHandler` / users エンドポイントの inline 401 / `CreateNavigationItemHandler`）を最初に参照していれば防げた。実装前に同系統の既存コードを確認するルーティンを徹底したい。

**フルスタック一気通貫の達成感**  
コメント機能は DB マイグレーションから OpenAPI 仕様、PHP ドメイン層、React エンティティ層・フィーチャー層・ページ層まで一人で完結させた。NENE2 パターンと FSD が軌道に乗っているため、新機能の「型」が掴めている感覚がある。
