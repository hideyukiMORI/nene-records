### 1. #121 画像フィールド + メディアアップロード API ✅

| 項目 | 内容 |
| --- | --- |
| Issue | [#121](https://github.com/hideyukiMORI/nene-records/issues/121) |
| PR | [#122](https://github.com/hideyukiMORI/nene-records/pull/122) — **OPEN** |

**Backend**

- `media` テーブル、`ServeMediaHandler`、`UploadMediaHandler` を実装
- `data_type = 'image'` / `'file'` を `FieldDefWriteValidator` に追加
- OpenAPI に `/api/v1/media/upload`・`/media/{year}/{month}/{filename}` を追加

**Frontend**

- `ImageFieldInput` の ESLint `no-confusing-void-expression` エラーを修正
- `FIELD_DATA_TYPES` enum を Prettier 整形

**Vite プロキシバグ修正（同梱）**

- **原因:** `vite.config.ts` のプロキシ先が `localhost:8080` ハードコード → 実際は `NENE_RECORDS_PORT=8082`、ポート 8080 は別アプリ（nene-corpus）だった
- **修正:** `loadEnv` でプロジェクトルートの `.env` を読み込み、`NENE_RECORDS_PORT` を動的解決
- **追加:** `/media` プロキシも欠落していたため追加
- **効果:** "Could not load site settings." エラーが解消

### 3. #125 ナビゲーション設定 CRUD ✅

| 項目 | 内容 |
| --- | --- |
| Issue | [#125](https://github.com/hideyukiMORI/nene-records/issues/125) |
| PR | [#124](https://github.com/hideyukiMORI/nene-records/pull/124) — **OPEN** |

**Backend（27 ファイル新規）**

- `navigation_items` テーブル（Phinx マイグレーション + SQLite テスト用 DDL）
- `src/NavigationItem/` に完全なドメイン層を実装
  - Entity / Repository（PDO 実装）/ 4 UseCases / 4 Handlers / ExceptionHandler / ServiceProvider / RouteRegistrar
- **エンドポイント:**
  - `GET /api/v1/navigation-items`（一覧、`display_order ASC`）
  - `POST /api/v1/navigation-items`（作成、201）
  - `PUT /api/v1/navigation-items/{id}`（更新）
  - `DELETE /api/v1/navigation-items/{id}`（削除、204）
  - `GET /api/v1/public/navigation-items`（認証不要、`ALWAYS_OPEN_PREFIXES` 対象）
- `AdminApiAuthMiddleware`: `/api/v1/navigation-items` を `ADMIN_ONLY_PREFIXES` に追加（GET も含め全メソッド保護）
- `CapabilityResolver`: ManageSettings（POST/PUT/DELETE）/ ReadSettings（GET）を追加
- `ApplicationServiceProvider`: `NavigationItemServiceProvider` を登録

**PHPStan で発見・修正したバグ 2 件:**
- `DeleteNavigationItemHandler` が `JsonResponseFactory::create(null, 204)` を使用 → `createEmpty(204)` に修正
- `NavigationItemNotFoundExceptionHandler::handle` の引数順が逆（`$request, $e` → `$exception, $request`）

**Frontend（FSD 準拠）**

| 層 | ファイル |

| Entity | `api-types.ts` / `model.ts` / `mapper.ts` / `query-keys.ts` / `queries.ts` / `mutations.ts` / `index.ts` |

| Page | `NavigationPage.tsx`（`/navigation`、`manage_settings` 権限チェック） |

- **AppShell**: 管理者のみ「ナビゲーション」リンクを表示
- **PublicShell**: パブリックサイトヘッダーに `usePublicNavigationItems()` からのリンクを表示
- **Router**: `/navigation` ルートを追加
- **i18n**: `en.ts` / `ja.ts` に `admin.navigation.*` / `admin.nav.navigation` を追加

## 課題・所感

- **Vite プロキシの落とし穴:** `.env` の `NENE_RECORDS_PORT` を読まずハードコードしていたため、ポートが異なる環境でサイレントに別コンテナへ接続していた。`loadEnv` を使うパターンを今後の標準にする。
- **PHPStan が頼りになった:** `DeleteNavigationItemHandler` の `null` 渡しと引数順の誤りを実装直後に検出。型チェックが省力化に直結している。
- **テストカバレッジ:** Navigation 層はハンドラー・リポジトリともに PHPUnit テストがまだない。M2 完了前に Integration テストを追加したい。
- **PR が溜まり始めた:** #112 / #120 / #122 / #123 / #124 の 5 本が OPEN。マージ優先度を整理してレビューを進める必要がある。
